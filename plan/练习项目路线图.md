> **原则**：每个项目独立可运行、有 README、有验收标准；后面的项目 **复用设计、沉淀接口、迁移成熟模块**，最终汇入旗舰项目 P6。

## P1：Model Gateway（第 1–2 周）

**一句话**：把大模型当不可靠外部服务封装。

**核心模块**：
```text
src/model_gateway/
  gateway.py          # 统一入口 ModelGateway
  adapters/
    deepseek.py         # 主模型（默认）
    moonshot.py         # Kimi / 月之暗面
    openai_compat.py    # 通义等 OpenAI 兼容接口
    openai_provider.py  # 官方 OpenAI（可选，后续启用）
  retry.py            # 指数退避
  ratelimit.py        # 令牌桶
  metrics.py          # token/耗时/费用
  schemas.py          # Pydantic 请求/响应
  tools/
    registry.py       # 工具注册（W2）
```

**CLI 验收**：
```bash
python -m model_gateway.cli chat "你好"
python -m model_gateway.cli bench --n 20   # 20 次调用报告
python -m model_gateway.cli tools calc "1+2*3"
```

**周验收（W1）**：20 次调用有模型、token、耗时、费用、错误记录。  
**周验收（W2）**：≥2 模型可切换；工具参数校验失败可捕获。

**迁移到 P6**：把稳定接口和测试迁入 `06-research-agent/packages/model_gateway/`，不要长期维护两份代码。

---

## P2：Mini RAG（第 3–5 周）

**一句话**：可引用、可拒答、可评测的迷你知识库问答。

**核心模块**：
```text
src/mini_rag/
  loaders/            # MD/PDF
  chunking.py
  index.py            # Chroma
  retriever.py        # 向量 + BM25 混合
  rerank.py
  qa.py               # 带引用回答 + 拒答
  eval/
    dataset.jsonl     # 50 题 → 扩到 100
    runner.py         # hit@k, MRR
```

**CLI 验收**：
```bash
python -m mini_rag.cli ingest ./data/docs
python -m mini_rag.cli ask "什么是 LangGraph？"
python -m mini_rag.cli eval --dataset eval/dataset.jsonl
```

**周验收（W5）**：hit@5 ≥ 80%；引用可追踪；有评测报告 markdown。

**分档目标**：
- **必达版**：Markdown + Chroma + 向量检索 + 引用回答 + 20 题评测。
- **进阶版**：PDF + hybrid search + rerank + 拒答 + 50 题评测。

**迁移到 P6**：把检索接口、评测脚本和 badcase 库迁入 `packages/rag/`，P2 保留为学习归档。

---

## P3：ReAct Agent（第 6 周）

**一句话**：手写 Think→Act→Observe，不用框架先理解本质。

**工具集（3 个）**：
- `search_kb` — 调用 P2 检索
- `read_file` — 读本地文件（沙箱目录）
- `calculator` — 安全计算

**核心**：`agent_loop.py`（max_steps、死循环检测、结构化 observation）

**验收**：
- [ ] 多步任务「查资料 → 计算 → 汇总」能跑通
- [ ] 能画图解释状态流转
- [ ] 有 5 条 badcase（死循环/错工具）及修复记录

**迁移到 P6**：保留为 `packages/agent_runtime/legacy_loop.py` 的参考实现，用于解释 Agent Loop 原理。

---

## P4：LangGraph Agent（第 7 周）

**一句话**：有状态、可中断恢复、带记忆的 Agent。

**核心**：
- StateGraph + Checkpointer（SQLite）
- 短期会话记忆 + 长期向量记忆（可选 Chroma）
- 流式输出

**验收**：
- [ ] 长任务中途 kill 进程后可 `--resume thread_id` 继续
- [ ] 能口述工作记忆 vs 长期记忆区别

**迁移到 P6**：把状态定义、Checkpointer、恢复流程沉淀到 `packages/agent_runtime/graph.py`。

---

## P5：MCP Tool Server（第 8 周）

**一句话**：用 MCP 标准协议暴露 P2+P3 能力，并加安全层。

**暴露工具**：
- `kb_search` / `kb_ingest`
- `agent_run`（受权限控制）
- `approve_action`（人工审批占位）

**验收**：
- [ ] 最小版：Cursor / MCP Inspector / 自写 Client 能 `tools/list`
- [ ] 最小版：`calculator` 或 `kb_search` 至少 1 个工具可被调用
- [ ] 最小版：prompt injection 测试 ≥3 条有记录
- [ ] 进阶版：`agent_run` + approval 占位流程
- [ ] 进阶版：Skills 注册表有 JSON Schema 文档

**迁移到 P6**：把最小 MCP Server 和工具 Schema 迁入 `packages/mcp_server/`，P5 保留协议实验记录。

---

## P6：Research & Execution Agent（第 9–15 周，旗舰）

**一句话**：面向知识工作的研究执行平台 — 面试主项目。

**模块沉淀路径**：
```text
P1 model_gateway  → packages/model_gateway/       # 迁移稳定接口和测试
P2 mini_rag       → packages/rag/                 # 迁移检索、引用、评测
P3/P4 agent       → packages/agent_runtime/       # 迁移状态、恢复、工具循环
P5 mcp_server     → packages/mcp_server/          # 迁移 MCP 工具暴露
新增              → packages/tools/ skills/ evaluation/ observability/
应用层            → apps/api/ apps/cli/ apps/web(可选)
```

**MVP 命令（W12 前必须全通）**：
```text
kb.ingest | kb.search | agent.ask | agent.plan | agent.run
agent.report | eval.run | trace.show
```

**生产补强（W13–15）**：100 条 golden、CI、Langfuse、安全报告、Docker Compose。

**作品集（W16）**：README + 架构图 + 10 分钟演示视频 + 2 篇博客。
