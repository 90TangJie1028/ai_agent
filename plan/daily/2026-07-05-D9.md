## 5 条笔记

1. **三条路径别混**：① prompt 拼 schema ② `json_object` ③ `json_schema`——约束递增，**都要 Pydantic 二次校验**
2. **传输 vs 结构化分层**：Adapter 只透传 `response_format`；`structured_output.py` + `chat_structured` 负责拼 prompt / parse / validate
3. **聚合 vs 多态**：行为分叉（Mock vs HTTP）→ Protocol；通用零件（RetryPolicy）→ 组合；只变数字 → config
4. **mock 实验表**：prompt 0% / json_object 100% / json_schema 100%（mock 在 `response_format` 时返回合法 JSON）
5. **校验失败重试**：概念已学，实现留待需要时加在 Gateway（不在 adapter）

## 学习走查（30m）

### 三种路径对照

| 路径 | API 参数 | 约束什么 | 仍可能失败 |
|:---|:---|:---|:---|
| A prompt-only | 无 | 靠模型听话 | markdown、缺字段、类型错 |
| B JSON mode | `response_format: {type: json_object}` | 合法 JSON 对象 | 字段/shape 不符 |
| C Structured Outputs | `response_format: {type: json_schema, ...}` | 生成期按 schema | 厂商不支持 |
