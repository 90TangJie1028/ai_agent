# 结构化输出（D8 速查）

W2 从 `@dataclass` 观测（`CallRecord`）升级到 **Pydantic 业务校验**：模型返回的是脏 JSON，必须 parse + validate + 可重试。

## 两层模型，别混

```text
chat(message)           → GatewayResult → CallRecord（观测，不校验外部）
chat_structured(...)    → StructuredAnswer（业务，Pydantic 校验 LLM JSON）
```

| 层 | 类型 | 类比 TS |
|:---|:---|:---|
| 观测 | `@dataclass CallRecord` | 内部日志 struct |
| 业务 | `BaseModel StructuredAnswer` | `zod.parse()` |

## Pydantic 三板斧

| 方法 | 作用 | TS 类比 |
|:---|:---|:---|
| `Model(...)` / `model_validate(dict)` | 构造 + 校验 | `schema.parse()` |
| `model_dump()` / `model_dump_json()` | 导出 | 对象 / `JSON.stringify` |
| `model_json_schema()` | JSON Schema | 给 `response_format` / tools |

## Field 常见写法

```python
answer: str = Field(..., description="...")           # 必填：... 是 Ellipsis，不是 spread
confidence: float = Field(ge=0, le=1)                 # 范围在赋值时检查
sources: list[str] = Field(default_factory=list)      # 勿写 = []（共享可变默认值）
provider: str | None = None                           # 可选
```

## chat_structured 四步

```text
① ChatRequest(...)              验入参
② prompt + model_json_schema()  告诉模型输出形状（不是 Pydantic 校验）
③ self.chat(prompt)             调模型，拿纯文本
④ json.loads + model_validate   验出参；失败 CallRecord.from_error
```

**要点**：`model_json_schema()` 声明一次，校验与 API schema 同源，避免手写两份定义漂移。
