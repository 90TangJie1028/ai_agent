# 结构化输出（D8–D9 速查）

W2 从 `@dataclass` 观测（`CallRecord`）升级到 **Pydantic 业务校验**：模型返回的是脏 JSON，必须 parse + validate + 可重试。

## 两层模型，别混

```text
chat(message)           → GatewayResult → CallRecord（观测，不校验外部）
chat_structured(...)    → StructuredAnswer（业务，Pydantic 校验 LLM JSON）
```

| 层 | 类型 | 类比 TS |
|:---|:---|:---|
| 观测 | `@dataclass CallRecord` | 内部日志 struct |
| 业务 | `BaseModel StructuredAnswer` | `zod.parse()` |

## 三条输出路径（D9）

| mode | API | 约束 | 校验落点 |
|:---|:---|:---|:---|
| `prompt` | 无 | prompt 贴 schema | Gateway `parse_structured_response` |
| `json_object` | `response_format: {type: json_object}` | 合法 JSON | 同上 |
| `json_schema` | `response_format: {type: json_schema, ...}` | 生成期 shape | 同上 |

**铁律**：无论哪条路径，Adapter 只透传 `response_format`；**Pydantic 校验在 Gateway / `structured_output.py`**。

## Pydantic 三板斧

| 方法 | 作用 | TS 类比 |
|:---|:---|:---|
| `Model(...)` / `model_validate(dict)` | 构造 + 校验 | `schema.parse()` |
| `model_dump()` / `model_dump_json()` | 导出 | 对象 / `JSON.stringify` |
| `model_json_schema()` | JSON Schema | 给 `response_format` / tools |

## chat_structured 流程

```text
① ChatRequest(...)                    验入参
② build_structured_prompt + build_response_format   按 mode 拼 prompt / API 参数
③ self.chat(..., response_format=...)  Adapter 发包收包
④ parse_structured_response           json.loads + model_validate
```
