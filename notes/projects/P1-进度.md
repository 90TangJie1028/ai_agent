## 已知坑

- DeepSeek 空 prompt 不一定失败；bench 真实 API 用无效 model 注入失败
- 默认 bench provider 是 `mock`，CI/离线友好
- `GatewayResult` 与裸 `str` 不兼容——旧代码需改 `.result.content`
- `arguments` 是 JSON **字符串**；必须 `validate_args` 后再进 handler
- 同步 `chat` / `chat_with_tools` 不走 Semaphore；并发闸门只约束 `async_chat`
- Moonshot 模型名以 `/v1/models` 为准；`moonshot-v1-8k` 会 `NotFoundError` → 用 `kimi-k2.6`
